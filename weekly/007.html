Tips & Tricks
Try effort control before you blame the model · Anthropic
Opus 4.8 ships a per-response effort dial. Crank it up when a one-shot answer keeps drifting. Drop it for quick rewrites and you spend rate limits slower. The model picks how hard to think, not how hard to talk. Effort control →
Add the word "workflow" to your prompt · Claude Code docs
Drop "workflow" anywhere in the ask and Claude Code writes a JavaScript orchestrator instead of doing the work inline. The script spawns subagents in parallel, runs in the background, and your session stays responsive. Use it for codebase audits and migrations. Requires v2.1.154+. Dynamic workflows →
Build the CLAUDE.md before you build the prompt · Power user guide
Stack, naming, folder layout, and explicit "do NOT" rules in CLAUDE.md make Claude Code 3 to 5x more accurate first try. Every time Claude does something wrong, add the rule. The file is your bug log turned into context. Power user guide →
Honesty is a prompting feature now · Anthropic
Opus 4.8 is roughly 4x less likely than 4.7 to ship code with a quiet bug it spotted. Ask it what it is unsure about before you ship. It will tell you what it skipped, what it stubbed, and what it cannot verify. Useful even on tasks that look done. Simon's read →

Feature Spotlight
This week
Dynamic Workflows, when Claude writes its own orchestrator
A dynamic workflow is a JavaScript script that Claude writes to coordinate subagents at scale. You describe the task. Claude drafts the script. A runtime executes it in the background while your chat stays responsive. The script is rerunnable, so the second pass costs less than the first. Reach for it when one agent will not finish in one context, or when the work splits cleanly across files or repos. Codebase audits, large migrations, cross-checked research. Add the word workflow to a prompt and Claude Code starts writing one. Requires v2.1.154 and Opus 4.8 to feel like the demo.
