Tips & Tricks
Run /clear between unrelated tasks · Builder.io
Context is your scarcest resource. When you switch tasks, the old files and half-finished threads still sit in the window and pull Claude off track. Type /clear to wipe the slate. A fresh context beats a clever prompt almost every time. 50 field-tested tips →
Make Claude show its work · Claude Code best practices
Ask for evidence, not a status update. "Tests pass" is a claim. The actual test output is proof. Tell Claude to paste the command it ran and what came back, or a screenshot of the result. Reading the evidence is faster than re-running it yourself. Best practices →
Reach for /dataviz when you need a chart · Claude Code
New this week. The /dataviz skill hands Claude real design guidance for charts and dashboards, so the output stops looking like a default plot. Ask for the view. Let the skill handle color, scale, and layout. Good for reports and quick data checks. Changelog →
Wrap your context in XML tags · ChatPromptGenius
Claude is trained to respect XML. Put the task, the context, and the format in separate tags like <context> and <format>. Claude treats each block as its own instruction instead of blending them. More reliable than markdown headers or plain delimiters. Advanced prompting →

Feature Spotlight
This week
Extended Thinking, and the effort dial that runs it
Extended thinking lets Claude reason step by step before it answers. On the newest models it runs in adaptive mode. Claude sizes up each request and decides how much to think, and it can think between tool calls, which is what makes it good at agent work. The effort parameter is your dial. At high or max, Claude almost always thinks. Lower it and Claude skips the reasoning on easy prompts, so you pay for depth only when the task needs it. Sonnet 5 pushes the top of that dial to xhigh. Turn it up for planning, hard debugging, and multi-step math. Turn it down for lookups and boilerplate.
Adaptive thinking, when Claude decides how hard to think
The default mode on Opus 4.8 and Sonnet 5. Claude weighs each request and thinks only as much as it needs. It also turns on interleaved thinking, so Claude can reason between tool calls in an agent loop.
Claude Platform Docs
The effort parameter, your cost lever
Effort is soft guidance on how much Claude thinks. High and max lean into reasoning. Lower levels skip it on simple prompts. Pair it with max_tokens as a hard cap so long thinking never blows the budget.

How to prompt a thinking model
Thinking models want different prompts. Give high-level goals and room to reason, not a rigid step-by-step script. This page covers what to change once the model reasons on its own.
