Tips & Tricks
CLAUDE.md hygiene · Builder.io
Keep CLAUDE.md under 200 lines. Past that, instruction adherence drops and you burn context on every turn. Cover only commands, conventions, and the three or four files Claude must always read. Push everything else into linked docs you reference on demand. 50 Claude Code tips →
Subagents · XDA Developers
Delegate research to subagents, not the main loop. Spin one off to grep, read, and summarize. It returns a digest. Your main session keeps its context clean. Treat subagents like async coworkers. Cheap to start, expensive to ignore. Why subagents pay off →
Plan Mode default · Builder.io
Make Plan Mode the default for anything touching 3+ files. Shift+Tab twice. The plan is cheap. The wrong refactor is not. Community consensus after Code w/ Claude: the only reason to skip Plan Mode is a single-file edit or a read-only question. Official workflows →
Opus 4.7 tokenizer · platform.claude.com
Budget for up to 35% more tokens with Opus 4.7. The new tokenizer encodes text less densely than 4.6. Same prompt, same context window, fewer characters fit. Re-check your CLAUDE.md size and your long-doc pipelines before you ship anything 4.7-only. What's new in 4.7 →

Feature Spotlight
This week
MCP servers, Claude's universal adapter
Model Context Protocol is the open standard that lets Claude read and write external systems. Think USB-C for tools. One protocol, one config, anything pluggable. The week's launches make it concrete. Office GA, Claude for Legal, and Small Business all ship as MCP connector bundles. The community marketplace cleared 770+ MCP servers as of May 14. Use MCP when you need durable two-way access to a real system (Slack, Postgres, GitHub, your design tool). Use a Skill when the value is in prompt scaffolding and instructions. Most production agents now run with 3 to 6 MCPs and a handful of Skills.
Connect Claude Code to tools via MCP
The official guide. Config locations on macOS, Windows, and Linux. How to scope MCPs per project versus globally. The right pattern for secrets handling. Worth re-reading after the Office and Legal launches added so many new connector shapes.

Agent Teams, Subagents, and MCP: the 2026 playbook
The clearest explainer of how MCPs, Subagents, and Agent Teams compose. MCP is your tools layer. Subagents are scoped helpers in one session. Agent Teams is multi-session, multi-context coordination. Mismatching the three is the most common 2026 architecture bug.
